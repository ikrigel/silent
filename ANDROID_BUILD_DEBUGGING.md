# Android APK Build Debugging Guide

This guide documents commands and techniques for debugging Android/Kotlin build issues discovered during APK build process.

## Quick Diagnostic Commands

### 1. Compile Kotlin Only (No Full Build)
Isolate Kotlin compilation errors without running the full APK assembly:

**PowerShell:**
```powershell
cd android; .\gradlew app:compileReleaseKotlin --stacktrace --info
```

**Bash:**
```bash
cd android && ./gradlew app:compileReleaseKotlin --stacktrace --info
```

**Why use this:**
- **Fast feedback** — Only compiles Kotlin, skips resource processing, dexing, packaging
- **Clear error location** — Shows exact `.kt` file and line number of compilation errors
- **No APK needed** — Test changes without waiting for full build
- **Typical runtime** — ~30-60 seconds vs 2-5 minutes for full build

**What to look for in output:**
```
e: /path/to/File.kt: (42, 13): Unresolved reference: missingClass
w: /path/to/File.kt: (50, 5): Variable 'unused' is never used
```

Lines starting with `e:` are **errors**, `w:` are **warnings**. These appear ABOVE the long Java stack trace.

---

### 2. Full Release APK Build Locally
Complete build with all optimizations and signing:

**PowerShell:**
```powershell
cd android; .\gradlew clean assembleRelease
```

**Bash:**
```bash
cd android && ./gradlew clean assembleRelease
```

**Why use this:**
- Verifies the **complete build chain** works end-to-end
- Tests **Kotlin compilation**, **resource processing**, **dexing**, **packaging**
- APK output: `android/app/build/outputs/apk/release/app-release.apk`
- **Typical runtime** — 2-5 minutes depending on cache state

**What to look for:**
```
BUILD SUCCESSFUL in 4s
```

At the end means the full APK was created successfully.

---

### 3. Display Build Environment
Verify Java, Gradle, and Kotlin versions in current environment:

**PowerShell:**
```powershell
cd android; .\gradlew --version
java -version
```

**Bash:**
```bash
cd android && ./gradlew --version && java -version
```

**Example output:**
```
Gradle 8.14.3
Build time: 2025-11-21 12:34:56 UTC
Java version: 21.0.1

openjdk version "21.0.1" 2023-10-17
```

**Why check this:**
- Ensures **JDK version matches** (`gradle.jvm.version` in `gradle.properties`)
- Verifies **Gradle wrapper version** (should be 8.14.3+)
- Confirms **Kotlin plugin version** (1.9.24 or later)

---

### 4. View Gradle Properties
Check configured JVM target, optimizations, and Android settings:

**PowerShell:**
```powershell
cat android/gradle.properties
```

**Bash:**
```bash
cat android/gradle.properties
```

**Key lines to verify:**
```properties
gradle.jvm.version=21              # Should match Java version used for compilation
org.gradle.parallel=true           # Parallel builds enabled
org.gradle.caching=true            # Caching enabled
android.useAndroidX=true           # AndroidX support
android.enableJetifier=true        # Jetifier enabled for library compatibility
```

---

### 5. Check Build Gradle Files
Verify critical build configuration that caused issues:

**Check app module:**
```powershell
cat android/app/build.gradle | Select-String -Pattern "compileOptions|kotlinOptions|firebase|kotlin-stdlib"
```

**Bash:**
```bash
grep -E "compileOptions|kotlinOptions|firebase|kotlin-stdlib" android/app/build.gradle
```

**What to look for:**
```gradle
compileOptions {
    sourceCompatibility JavaVersion.VERSION_21  // Must match Kotlin jvmTarget
    targetCompatibility JavaVersion.VERSION_21
}

kotlinOptions {
    jvmTarget = "21"  // Must match Java version
}

implementation 'com.google.firebase:firebase-auth:23.1.0'  // Pinned version prevents unwanted upgrades
```

---

### 6. Clear Gradle Cache (When Stuck)
Remove all cached build artifacts to force fresh build:

**PowerShell:**
```powershell
Remove-Item -Recurse -Force android/.gradle
Remove-Item -Recurse -Force android/app/build
Remove-Item -Recurse -Force android/build
```

**Bash:**
```bash
rm -rf android/.gradle android/app/build android/build
```

**Why do this:**
- **Stale metadata** — Old Kotlin/Firebase version info cached
- **Corruption** — Downloaded artifacts may be incomplete or corrupt
- **Force fresh resolution** — Gradle re-downloads and re-validates all dependencies
- **Last resort** — When incremental build stuck or behaving oddly

**Warning:** This adds 2-3 minutes to next build (re-downloads everything).

---

### 7. View Build Reports
Access detailed build issue reports generated by Gradle:

**Problems report (HTML):**
```
android/build/reports/problems/problems-report.html
```

**Open in browser (Windows PowerShell):**
```powershell
Invoke-Item android/build/reports/problems/problems-report.html
```

**Open in bash:**
```bash
open android/build/reports/problems/problems-report.html  # macOS
xdg-open android/build/reports/problems/problems-report.html  # Linux
start android/build/reports/problems/problems-report.html  # Windows Git Bash
```

**What it contains:**
- Detailed error messages from failed tasks
- Line numbers and file paths
- Dependency resolution issues
- Configuration warnings

## Common Issues & Solutions Found

### Issue: JVM Target Mismatch
**Error:** `Inconsistent JVM-target compatibility detected for tasks 'compileReleaseJavaWithJavac' (17) and 'compileReleaseKotlin' (21)`

**Root causes found:**
1. `gradle.properties` has `gradle.jvm.version=17` (Gradle daemon runs on Java 17)
2. `app/build.gradle` specifies `JavaVersion.VERSION_21` (compilation target is 21)
3. `capacitor.build.gradle` (auto-generated) had `JavaVersion.VERSION_17`
4. Kotlin `jvmTarget = "21"` doesn't match Java target

**Fix:**
- Update `gradle.properties`: `gradle.jvm.version=21`
- Update `app/build.gradle`: `sourceCompatibility JavaVersion.VERSION_21`
- Update `capacitor.build.gradle`: `sourceCompatibility JavaVersion.VERSION_21`
- Ensure Kotlin target matches: `jvmTarget = "21"`

---

### Issue: Kotlin Metadata Mismatch
**Error:** `Module was compiled with an incompatible version of Kotlin. The binary version of its metadata is 2.1.0, expected version is 1.9.0`

**Root cause found:**
- Firebase authentication plugin was pulling `firebase-auth:24.0.1` (compiled with Kotlin 2.1.0)
- Project only had Kotlin 1.9.24 compiler
- Forced Kotlin stdlib resolution didn't fix metadata compatibility

**Fix:**
- Pin Firebase Auth version explicitly: `implementation 'com.google.firebase:firebase-auth:23.1.0'`
- Remove harmful force resolution block from `build.gradle`
- Ensure Firebase BOM compatible with Kotlin version (BOM 32.7.0 with Kotlin 1.9.24)

---

### Issue: Gradle Cache Corruption
**Error:** Builds work locally but fail in CI with metadata errors

**Root cause found:**
- CI reuses Gradle cache from previous failed builds
- Stale metadata from old Firebase/Kotlin versions cached
- Fresh checkout but `.gradle` directory retained old state

**Fix:**
- Clear cache before build: `rm -rf android/.gradle android/app/build`
- Add cache-clearing step to CI workflow
- Each CI run starts fresh without inherited cache

---

### Issue: Capacitor Auto-Generated File Override
**Problem:** `capacitor.build.gradle` auto-generated by `capacitor update` would override manual changes

**Root cause found:**
- Capacitor CLI generates this file from template on `npx cap sync`
- Manual Java version updates in this file get overwritten
- Better to fix in main `app/build.gradle` or document the issue

**Solution:**
- Make critical changes in `app/build.gradle` (not auto-generated)
- Document that `capacitor.build.gradle` needs manual updates after `capacitor update`
- Consider upstream contribution to Capacitor Android template for Java 21 support

---

## Workflow: Debugging New Build Error

1. **Isolate the error** — Run `./gradlew app:compileReleaseKotlin --stacktrace --info`
2. **Find the real error** — Scroll ABOVE the stack trace, look for `e:` lines with file + line number
3. **Check environment** — Run `./gradlew --version && java -version`
4. **Verify config files:**
   - `gradle.properties` (JVM version)
   - `app/build.gradle` (Java/Kotlin targets, dependencies)
   - `capacitor.build.gradle` (Java version)
   - Root `build.gradle` (Kotlin plugin, resolutionStrategy)
5. **Test locally** — `./gradlew clean assembleRelease`
6. **Clear if stuck** — `rm -rf android/.gradle android/app/build android/build`
7. **Check reports** — Open `android/build/reports/problems/problems-report.html`
8. **Document findings** — Update CLAUDE.md with what you learned

## Android Build Stack

| Layer | Tool | Version | Config File |
|-------|------|---------|-------------|
| **JVM Runtime** | Java/Temurin | 21 | `gradle.properties` (`gradle.jvm.version`) |
| **Build System** | Gradle | 8.14.3 | `gradle/wrapper/gradle-wrapper.properties` |
| **Android Plugin** | AGP | 8.13.0 | `android/build.gradle` (`classpath`) |
| **Kotlin Compiler** | Kotlin | 1.9.24 | `android/build.gradle` (`kotlin-gradle-plugin`) |
| **Compile Target** | Java/Kotlin | 21 | `app/build.gradle` (`compileOptions`, `kotlinOptions`) |
| **SDK Version** | Android SDK | 36 | `android/variables.gradle` (`compileSdkVersion`) |
| **Firebase** | Firebase Auth | 23.1.0 | `app/build.gradle` (explicit pin) |

All layers must be **consistent** — mismatches cause metadata/target errors.
